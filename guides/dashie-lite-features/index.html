Dashie Lite

Dashie Lite Complete Guide

Last updated: January 2025

Dashie Lite is a free, lightweight Android app designed for hosting Home Assistant dashboards. It provides a clean, full-screen experience with helpful extras like a sidebar for quick access, customizable settings, and optional voice control with the "Hey Dashie" wake word.

In This Guide

  • Getting Started
  • Using the Sidebar
  • Voice Control (Upgrade)

Getting Started

The first step to setting up Home Assistant on Dashie Lite is to configure your URL. There are 2 options to do this:

  1. Enter the URL for your dashboard including parameters to hide unwanted items
  2. Autobuild URL: Dashie Lite looks for your HA instance on the local network. If it doesn't find it, enter your HA base URL manually, the dashboard name, and select which items you'd like to hide (such as the sidebar, search, and assistant).

After that, click "Launch Dashboard" and you'll be taken to a Home Assistant Login screen. Note, this is not the native login screen on Fire tablets since the animations on that page interfere with the Fire Tablet's keypad.

Once logged in, your Home Assistant dashboard will display full-screen. Swipe from the left edge to reveal the sidebar.

Using the Sidebar

The sidebar provides quick access to essential features without leaving your dashboard. Swipe from the left edge of the screen to reveal it.

Audio

If Voice is enabled, this section allows you to set the volume for HA's responses, mute them, or mute your microphone.

Voice

You can enable voice control here to send voice commands to HA using "Hey Dashie" as the wake word. Responses are read aloud (unless you mute the audio) and displayed on the screen, if the display responses option is enabled.

Screen

Dashie Lite can be set to keep your screen on, if you disable the Fire tablet's screen saver. You can set a timeout for the screen to dim. Once dim, the screen can be woken up by touching the screen or with motion detection. Optimal motion detection is done with the camera, which requires camera permission. If the camera is not enabled, the brightness sensor will be used.

System

Dashie Lite supports the Fully Kiosk API commands, which can be enabled here. You can set Dashie Lite to load on boot, though the latest Fire tablets require a swipe to start. There is also the option to stop the status (top) and nav (bottom) bars from being shown with a swipe.

Voice Control (Upgrade)

Voice control with "Hey Dashie" wake word detection is available as a $12 upgrade. Once enabled, you can control your Home Assistant devices hands-free.

Voice control features:

  • Local wake word detection - "Hey Dashie" is processed on-device for speed
  • Home Assistant integration - Uses HA's Assist pipeline for command processing
  • Visual feedback - On-screen indicators show listening and processing states

For setup instructions, see our Voice Control Setup Guide.

Using the API

Dashie Lite includes a Fully Kiosk-compatible REST API that runs on port 2323. This enables Home Assistant's Fully Kiosk integration to control your Android device directly.

Enabling the API

In the sidebar under System, toggle on "Enable Fully Kiosk API". You can optionally set a password for authentication. Once enabled, the API is accessible at:

http://<device-ip>:2323/?cmd=<command>&password=<your-password>

Connecting to Home Assistant

To add Dashie Lite to Home Assistant using the Fully Kiosk integration:

  1. Go to Settings → Devices & Services → Add Integration
  2. Search for "Fully Kiosk Browser"
  3. Enter your device's IP address
  4. Set the port to 2323
  5. Enter your API password (leave blank if none set)

Supported Commands

Dashie Lite supports the most commonly-used Fully Kiosk API commands:

Command Description
deviceInfo Returns device state (battery, screen, network, etc.) - polled by HA every 30-60s
screenOn Turn screen on
screenOff Turn screen off
setStringSetting Set configuration (supports screenBrightness 0-255)
setAudioVolume Set volume (level 0-100, stream 1-5)
textToSpeech Speak text using Android TTS
stopTextToSpeech Stop TTS playback
loadUrl Navigate to a specific URL
loadStartUrl Reload the configured start URL
restartApp Restart Dashie Lite
toForeground Bring app to foreground
startScreensaver Dim the screen (enter screensaver mode)
stopScreensaver Wake the screen (exit screensaver mode)
triggerMotion Simulate motion detection (resets sleep timer)
playSound Play audio from a URL
stopSound Stop audio playback
getCamshot Capture image from camera (returns JPEG)
startApplication Launch another app by package name
setOverlayMessage Show a toast/overlay message
clearCache Clear WebView cache
clearWebstorage Clear WebView local storage
lockKiosk Lock kiosk mode (hide sidebar controls)
unlockKiosk Unlock kiosk mode (show sidebar controls)

Example API Calls

You can test the API directly from your browser or with curl:

# Get device info
curl "http://192.168.1.100:2323/?cmd=deviceInfo&type=json&password=mypass"

# Turn screen on
curl "http://192.168.1.100:2323/?cmd=screenOn&password=mypass"

# Set brightness to 50%
curl "http://192.168.1.100:2323/?cmd=setStringSetting&key=screenBrightness&value=128&password=mypass"

# Speak a message
curl "http://192.168.1.100:2323/?cmd=textToSpeech&text=Hello%20world&password=mypass"

Tip

The deviceInfo command returns battery level, screen state, WiFi signal, storage info, and more. Home Assistant polls this automatically when using the Fully Kiosk integration.

Tips & Tricks

Stay Awake Setting

In Developer Options, enable "Stay Awake" to prevent screen sleep while charging. This keeps your dashboard always visible when the tablet is plugged in.

Auto-Launch on Boot (ADB Commands)

These ADB commands allow Dashie Lite to launch automatically on boot without requiring user intervention. Connect your tablet via USB and run these commands from your computer:

# Grant overlay permission (required for background activity starts)
adb shell appops set com.dashieapp.Dashie.halite SYSTEM_ALERT_WINDOW allow

# Disable lock screen ads (Amazon Special Offers)
adb shell settings put global LOCKSCREEN_AD_ENABLED 0

# Disable lock screen entirely (no swipe to unlock)
adb shell locksettings set-disabled true

Note

Even with these settings enabled, the app will not start until the Fire tablet completes its system loading process, which takes approximately 60 seconds after power on.
